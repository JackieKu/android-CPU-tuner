Profiles

Profiles define settings that are applied by the triggers.

There are two types of settings:

  • CPU settings
    These setting determine the power consumption and the responsiveness of the CPU.
  • Services
    These setting allow you to switch services (like mobiledata, syncing, wifi) which consume a lot of power.

    CPU settings

    The CPU tuning part of CPU tuner is a front end to the linux kernel cpufreq interface.

    Services

    CPU tuner allows you to switch the state of battery consuming services.
    You can choose between three options:

  • Leave unchanged
    The state of this service will not be changed by this profile.
  • Enable
    The service will be enabled.
  • Disable
    The service will be disabled.

    • In case you enable Manual service changes in settings/system changes of service states will only happen if the state is not changed for outside of CPU tuner or if a new trigger is started.
    E.g. if a profile of a trigger switches background sync to off and you switch it back on, it will stay on until a new trigger activates a profile that siwtches it again.
